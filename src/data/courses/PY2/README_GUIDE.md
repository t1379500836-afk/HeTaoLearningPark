```json
{

  "resources": {
    "flashcard": {
      "file": "src\\data\\courses\\PY2\\L7\\L7-1\\flashcard.png",
      "content": "OCR提取的文字内容..."
    },
    "knowledge": { ... },
    "lecture": { ... },
    "exercise": { ... },
    "solution": { ... }
  }
}
```

### README.md
由脚本自动生成的 Markdown 格式内容，用于人工校对。

**注意**：重新运行 `extract_content.py` 会覆盖此文件。

## 内容筛选规则（重要）

原始学习资料包含与核桃编程客户端相关的内容（剧情动画、游戏角色等）。
在 Web 端展示时，需要筛选掉这些内容，只保留纯编程知识。

### 保留的内容

| 类型 | 示例 |
|------|------|
| 编程语法 | `split命令用于分割字符串` |
| 函数说明 | `print()函数用于输出内容` |
| 代码示例 | `for i in range(10):` |
| 英语单词 | `encode: 编码` |
| 编程练习题 | 选择题、代码题 |

### 舍去的内容

| 类型 | 示例 | 原因 |
|------|------|------|
| 剧情描述 | `在核桃部落的奇妙森林里...` | 客户端剧情 |
| 游戏角色 | `小猴子、蝴蝶、核桃队长` | 客户端角色 |
| 场景描述 | `帮助蝴蝶找到回家的路` | 客户端游戏场景 |
| 项目日志 | `项目进度：已完成 50%` | 客户端进度 |

### 灰色内容（需判断）

| 类型 | 判断标准 |
|------|---------|
| 科学探索 | 与编程知识相关则保留 |
| 趣味知识 | 有助于理解编程概念则保留 |

## 人工校对指南

校对自动生成的 README.md 时，请按以下步骤：

### 1. 单词卡 (flashcard)
- ✅ 保留：英语单词、音标、中文释义、例句
- ❌ 删除：无关的装饰文字

### 2. 知识点图 (knowledge)
- ✅ 保留：编程语法、命令说明、代码示例
- ❌ 删除：剧情背景描述

### 3. 讲义 (lecture)
- ✅ 保留：编程概念、语法讲解、代码示例
- ⚠️ 判断：科学探索（与编程相关则保留）
- ❌ 删除：剧情引入、项目日志

### 4. 课后习题 (exercise)
- ✅ 保留：所有编程练习题
- ⚠️ 修改：删除题目中的剧情背景描述，保留核心编程问题

**示例**：
```
原题：在核桃部落的奇妙森林里，每个小动物都有一个编号...
修改：给定一个二维列表 animals，通过索引获取指定位置的元素...
```

### 5. 习题解析 (solution)
- ✅ 保留：答案、解题思路、代码解析
- ❌ 删除：无关的剧情描述

## 课程命名规范

创建或更新 `lesson-data.js` 文件时，需遵循以下命名规范。

### 课程标题 (title)

从 lecture 文档第一页提取，格式为 `模块X：模块名 课程名 LXX-X`：
- 去掉模块名和课次编号
- 仅保留课程名称部分

**示例**：
| Lecture 原文 | title |
|-------------|-------|
| 模块1：split命令原始森林 L7-1 | 字符串的奇妙旅行 |
| 模块3：查找思想智取核桃山 L7-2 | 智取核桃山 |
| 模块5：列表排序忍无可忍 L7-3 | 忍无可忍 |

### 课程副标题 (subtitle)

使用"学会..."开头，简洁描述本节课核心知识点（2-4个知识点）。

**格式**：`学会[知识点1]和[知识点2]`

**示例**：
| 课程 | subtitle |
|------|----------|
| L7-1 | 学会遍历和分割字符串 |
| L7-2 | 学会查找最大最小值和列表推导式 |
| L7-3 | 学会列表排序和求和 |

### 注意事项

- title 必须与 lecture 文档一致
- subtitle 应简洁明了，控制在15字以内
- 多个知识点用"和"连接，不使用顿号或逗号

### 单元详情页卡片显示规范

**文件**：`src/views/UnitView.vue`

单元详情页（Level 7 详情页）的课程卡片必须从 `lesson-data.js` 动态加载数据：

| 显示元素 | 数据来源 |
|---------|---------|
| 卡片标题 | `lessonMeta.title` |
| 卡片描述 | `lessonMeta.subtitle` |
| 底部标签 | 固定显示：📖 知识点、⌨️ 打字练习、✏️ 课后习题 |

**实现要求**：
- 使用动态 `import()` 加载每节课的 `lesson-data.js`
- 如果数据加载失败，使用默认值兜底
- 卡片显示与课程详情页（LessonView）完全对应一致

## 更新 lessons.config.js

校对完成后，需要更新配置文件：

1. 确认课次标题（从讲义中提取实际主题）
2. 确认课次描述（简述编程知识点）
3. 更新资源路径
4. 标记课次状态

**配置文件位置**：[lessons.config.js](lessons.config.js)
